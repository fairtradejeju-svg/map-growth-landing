📋 MAP Growth 개요
MAP Growth가 무엇이고, 어떤 과정으로 운영되는지 전체 흐름을 안내합니다.
▶️ 영상으로 보기: MAP Introduction Video (YouTube)
MAP Growth란?

MAP Growth는 NWEA(현 HMH 산하)가 개발한 컴퓨터 적응형(Computer-Adaptive) 학업성취도 평가입니다. K–12 학생의 Reading, Mathematics, Language Usage, General Science 영역을 측정하며, 학생 수준에 맞게 문제 난이도가 자동 조절됩니다.

💡 결과는 RIT 점수(Rasch Unit)로 표시되며, 학년에 무관한 동일 척도로 성장을 추적할 수 있습니다.
운영 전체 흐름
  1. 로스터링 — 학생·교사 정보를 플랫폼에 등록 (매 시험 기간 전)
  2. 시험 환경 설정 — 기기 점검, 보안 브라우저 설치, 시험 일정 확인
  3. 시험 진행 — 감독관(Proctor)이 세션 생성 → 학생 로그인 → 시험 시작
  4. 리포트 확인 — 시험 완료 후 학생·학급·학교 레벨 리포트 분석
  5. 교수 활용 — 리포트 데이터 기반으로 학습 지도 및 학부모 상담
시험 시기 (일반적 기준)
시기명칭목적
가을 (9–10월)Fall Testing기초선(Baseline) 측정
겨울 (1–2월)Winter Testing중간 진도 점검
봄 (4–6월)Spring Testing연간 성장 측정
📌 한국 학원의 경우 시험 시기를 자체적으로 설정할 수 있습니다. 학원 일정에 맞추어 District Assessment Coordinator가 Testing Window를 조정합니다.
👤 역할과 권한
MAP 플랫폼에서 각 사용자에게 부여되는 역할(Role)과 해당 권한을 설명합니다.
역할한국 학원 대응주요 권한
System Administrator 원장 / IT 담당 모든 설정 접근, 사용자 관리, 시스템 환경설정
Data Administrator 원장 / 행정 담당 로스터 업로드, 학생·교사 데이터 관리
District Assessment Coordinator 원장 / 시험 총괄 시험 일정 설정, 시험 제한사항 관리, 전체 리포트 접근
Proctor 감독 선생님 시험 세션 생성·관리, 학생 확인(Confirm), 일시정지/중단

💡 한 사람에게 여러 역할을 부여할 수 있습니다. 소규모 학원에서는 원장이 System Admin + Data Admin + District Assessment Coordinator를 겸하는 것이 일반적입니다.
📝 1. 로스터링 개요
로스터링이란 학생과 교사 정보를 MAP 플랫폼에 등록하는 과정입니다. 시험과 리포트의 기반이 됩니다.
왜 로스터링이 중요한가?

로스터링은 단순히 이름을 등록하는 것이 아닙니다. 학생이 어떤 학급에 속하고, 어떤 선생님이 그 학급을 담당하는지를 결정합니다. 이 구조가 정확해야 시험 후 리포트가 올바르게 출력됩니다.

⚠️ 로스터링이 잘못되면 리포트에서 학생이 누락되거나, 선생님이 본인 학급 데이터에 접근할 수 없는 문제가 발생합니다.
로스터링 방법 2가지
방법설명적합한 경우
HMH Rostering Service (HRS) HMH 통합 로스터링 플랫폼. ClassLink/Clever 연동 지원 HMH 다른 제품도 사용하는 기관, 대규모 학교
NWEA Legacy Rostering CSV 템플릿 기반 수동 업로드 한국 학원 대부분 (추천)
✅ 한국 학원은 대부분 NWEA Legacy Rostering을 사용합니다. 아래 가이드는 이 방식을 기준으로 설명합니다.
로스터링 전체 흐름
  1. MAP 플랫폼에서 로스터 CSV 템플릿을 다운로드
  2. Excel에서 학생·교사 정보를 템플릿에 입력
  3. CSV(쉼표 구분) 형식으로 저장
  4. MAP 플랫폼에 업로드 (Import)
  5. 오류가 있으면 수정 후 재업로드
📄 1.1 로스터 준비
Data Administrator 역할이 필요합니다.
1 템플릿 다운로드
  1. start.mapnwea.org에 로그인
  2. 메인 메뉴에서 Rostering → Import Profiles/Rostering 클릭
  3. Access NWEA Legacy Rostering 선택
  4. Download Templates 섹션에서 템플릿 다운로드:
    • MAP Roster File Template — 메인 템플릿 (필수)
    • MAP Programs File Template — 프로그램 참여 학생용 (선택)
2 템플릿 구성

다운로드한 Excel 파일에는 2개 탭이 있습니다:

탭 이름용도필수 여부
StandardRoster 학생, 담당교사, 학급 정보 입력 ✅ 필수
Additional Users 관리자 등 추가 직원, 교사에게 추가 역할 부여 선택
3 CSV 저장 시 주의사항

입력을 완료한 후 반드시 CSV(쉼표 구분) 형식으로 저장해야 합니다.

🚨 CSV UTF-8로 저장하면 안 됩니다! Excel에서 저장할 때 CSV(쉼표 구분)(*.csv)를 선택하세요. CSV UTF-8은 오류를 유발합니다.
🌏 한국 Windows 사용자 주의: 한국어 Windows에서는 CSV 구분자가 쉼표(,) 대신 세미콜론(;)으로 설정되어 있을 수 있습니다.
해결: 제어판 → 시계 및 국가 → 지역 → 추가 설정 → 목록 구분 기호를 ,(쉼표)로, 소수점 기호를 .(마침표)로 변경한 후 Excel을 재시작하세요.

📥 1.2 로스터 가져오기 (Import)
Data Administrator 역할이 필요합니다.
▶️ 인터랙티브 교육: Import Your Roster 영상 가이드
1 Import 기본 절차
  1. start.mapnwea.org에 로그인
  2. Rostering → Import Profiles/Rostering 선택
  3. Access NWEA Legacy Rostering 클릭
  4. 설정 확인:
    • Roster Type: Standard
    • Term: 현재 학기 (기본값 유지)
    • Import 유형: Standard — Add/Update (권장)
  5. 준비한 CSV 파일 업로드
  6. Preview에서 데이터 확인 → 문제 없으면 Import 실행
Import 유형별 비교
유형동작사용 시기
Add/Update 신규 추가 + 기존 정보 업데이트 ✅ 대부분의 경우 이것을 사용
Overwrite 해당 학교의 기존 데이터를 모두 삭제 후 재등록 ⚠️ 학년 배정 오류 등 전면 수정 시에만
Students Only 학생 데이터만 추가 (학급 배정 없음) 긴급 학생 추가 시 (나중에 전체 import 필요)
Instructors Only 교사 데이터만 추가 (학급 배정 없음) 교사 계정 급하게 필요할 때
⚠️ Overwrite 주의: 로스터 파일에 포함되지 않은 학생/교사는 해당 학기에서 삭제됩니다. 반드시 전체 데이터가 포함된 파일로 실행하세요.
⏱️ Import 타이밍 팁
📌

• 한 기관(District)에서 동시에 하나의 Import만 실행 가능

• 시험 성수기에는 대기열에 들어가 최대 48시간 소요될 수 있음

• 대용량 파일은 여러 개로 나누어 업로드 가능

• Import 진행 중에는 다른 사용자가 프로필을 수정하지 않도록 안내

⚠️ 1.3 Import 오류 해결
로스터 Import 시 발생할 수 있는 오류 유형과 해결 방법입니다.

증상: 업로드 직후 "we have encountered an error with your file" 메시지

원인해결
CSV 형식이 잘못됨 (UTF-8로 저장)Excel에서 CSV(쉼표 구분)으로 재저장
컬럼/헤더가 템플릿과 다름NWEA 템플릿의 헤더 행을 복사하여 붙여넣기
불필요한 데이터가 포함됨데이터 범위만 선택 → 새 템플릿에 붙여넣기
한국어 Windows에서 구분자가 세미콜론제어판에서 목록 구분 기호를 쉼표로 변경
오류 메시지원인해결
"XXX cannot be the same for more than one YYY"서로 다른 사람이 같은 ID/사용자명을 공유오류 메시지의 Column Name, Row Number 확인 후 중복 제거
"Field cannot be empty"필수 컬럼에 빈 값해당 행과 열을 찾아 빠진 정보 입력
"Date of Birth must be a valid date"날짜 형식 오류MM/DD/YYYY 형식으로 통일
💡 팁: 오류가 많을 때는 해당 열을 필터링하여 문제 데이터만 골라 수정하면 빠릅니다.

이전에 등록된 데이터와 현재 업로드 데이터가 일치하지 않을 때 발생합니다.

학교명/학년 등 불일치:

예: 로스터에 "Foxcroft Elem"인데 MAP에는 "Foxcroft Elementary"로 등록된 경우

→ "Replace incoming XX with the following selection"을 선택하여 MAP에 있는 기존 이름으로 매칭

교사/학생 중복:

같은 사람이 여러 레코드로 매칭될 때 발생

→ 올바른 기존 프로필을 선택하거나, 사용자명/이메일을 수정하여 해결

🚫 피해야 할 선택: "Import users without creating associations" 또는 "Do not import incoming records"는 선택하지 마세요. 학생/교사가 리포트에서 누락됩니다.
⚙️ 2.1 시험 세션 만들기
Proctor 역할이 필요합니다.
▶️ 영상으로 보기: MAP Growth 로그인 방법
1 시험 세션 시작
  1. start.mapnwea.org에 로그인
  2. MAP Growth 선택
  3. Manage Test Sessions 클릭
  4. 아래 3가지 옵션 중 선택하여 세션 생성
▶ 영상: Find Students to Test

특정 학생 그룹을 미리 구성할 때 사용합니다.

  1. Find Students to Test 클릭
  2. 검색 조건(학교, 학년 등) 선택 → Search
  3. 검색 결과에서 Add Students
  4. 학생 수를 100명 이하로 유지 (너무 많으면 로그인 느려짐)
  5. 학생 선택 → Assign Test로 시험 배정
  6. 필요 시 Assign Accommodations로 편의사항 설정
  7. 나중에 시험할 경우 Save Session으로 저장
💡 세션 이름 팁: 공백, 특수문자, 대문자를 피하세요. 예: jbmath72 (이니셜+과목+학년+반) 또는 21eagle918 (교실+마스코트+날짜)
▶ 영상: Test My Class

본인이 담당하는 학급을 바로 시험할 때 사용합니다.

  1. Test My Class 클릭
  2. 학급이 여러 개면 하나를 선택
  3. Proctor 콘솔이 열리고 세션이 시작됨
  4. 학생 선택 → 시험 배정 및 편의사항 설정
⚠️ 버튼이 작동하지 않으면: 해당 학기에 교사 프로필이 학급에 연결되지 않은 것입니다. 학원 관리자에게 로스터 확인을 요청하세요.

이전에 저장해둔 세션을 불러와 시험을 진행합니다.

  1. Saved Testing Sessions 옆의 Show 클릭
  2. 다른 사람이 만든 세션을 보려면 Created By 필터 변경
  3. 세션 선택 → Test Now
📌 한 번에 하나의 세션만 실행할 수 있습니다. 이미 실행 중인 세션이 있으면 Return to Testing을 선택하세요.
▶️ 시험 배정 관련 영상
▶ 세션 편집 중 시험 배정 ▶ 활성 세션에서 시험 배정
🔑 2.2 학생 로그인
학생이 시험에 접속하는 과정을 단계별로 안내합니다.
A 학생 도착 전 — 감독관 준비
  1. Proctor 콘솔에서 Test Students 페이지 확인
  2. 화면 상단의 Session Name과 Password 확인
  3. 세션 이름과 비밀번호를 칠판이나 화면에 표시
📌 비밀번호는 세션을 시작할 때마다 새로 생성되며, 당일 밤에 만료됩니다.
B 학생 로그인 절차
  1. 학생 기기에서 NWEA 보안 테스트 브라우저/앱 실행
    또는 브라우저에서 test.mapnwea.org 접속
  2. 로그인 화면에서 Session Name과 Password 입력
  3. 학생 이름 확인 및 시험 선택 등 화면 안내에 따라 진행
🎮 연습 시험: 첫 시험 전에 학생들에게 연습 시험을 권장합니다. 로그인 화면에서 Try the Practice Test 선택 → Username/Password 모두 grow 입력
C 학생 확인 (Confirm)
▶ 영상: 학생 Confirm 방법
  1. 학생이 로그인하면 Proctor 콘솔에 "To Be Confirmed" 상태로 표시
  2. Refresh Page를 눌러 상태 갱신
  3. Confirm 버튼으로 일괄 확인 (또는 개별 학생 선택 → Select Action → Confirm)
  4. 학생 화면에서 Start Test 클릭하면 시험 시작
💡 인터넷이 느린 경우, 학생들이 한꺼번에 시작하지 말고 몇 명씩 순차적으로 시작하게 하세요.

⚡ Rapid Guessing (빠른 추측) 알림

학생이 문제를 제대로 읽지 않고 빠르게 답을 선택하면 시험이 자동으로 일시정지되고, Proctor 콘솔에 알림이 표시됩니다.

  1. Proctor 콘솔 상단의 Proctor PIN 확인
  2. 해당 학생에게 다가가서 신중히 답하도록 격려
  3. 학생 기기에서 PIN 입력 → 시험 재개
⚠️ 반복적으로 Rapid Guessing이 발생하면 (3회 일시정지) 해당 학생의 시험을 Suspend하고 다른 날 다시 진행하는 것을 권장합니다.
🎙️ 시험 전 학생 안내 스크립트

이 시험에서 최선을 다해주세요. 여러분이 무엇을 잘하고, 어디를 더 도와줄 수 있는지 알기 위한 시험입니다.

시간제한은 없으니 각 문제를 천천히 읽고 답해주세요.

쉬운 문제도 있고, 어려운 문제도 있습니다. 모든 답을 알 필요는 없어요.

확실하지 않은 문제는 분명히 틀린 답을 먼저 제외하고, 나머지에서 선택하세요.

다음 문제로 넘어가면 이전 문제로 돌아갈 수 없으니 신중하게 답하세요.

시험 중 문제가 생기면 손을 들어주세요.

🏁 2.4 시험 종료
시험 세션을 마무리하는 절차입니다.
전체 종료
  1. Proctor 콘솔에서 End Testing Session 클릭
  2. 아직 시험 중인 학생이 있으면 해당 시험은 Suspend 처리된다는 안내 확인 → Yes
  3. 선택:
    • Save and Exit — 같은 학생 그룹으로 나중에 다시 시험할 경우
    • Delete Session — 다른 그룹을 구성할 경우 (시험 결과에는 영향 없음)
  4. 학생 기기에서 브라우저/앱 종료:
    • PC/Mac: 상단 X 버튼
    • iPad: 홈 버튼
    • Chromebook: X 또는 Shift+Alt+K
📌 매일 밤 모든 세션은 자동 종료됩니다. 미완료 시험은 Suspend 상태가 됩니다.
한 학생의 시험만 교체
  1. 학생 선택 → Test Again
  2. Assign Test로 새 시험 배정
  3. 학생에게 같은 세션 이름과 비밀번호로 다시 로그인 안내

🔧 트러블슈팅
시험 중 자주 발생하는 문제와 해결 방법을 모았습니다. 클릭하면 답이 열립니다.
❓ 학생이 로그인했는데 목록에 안 보여요
학생의 상태를 확인하세요:
• Confirmed 상태: Select Action → Do Not Confirm 선택 후 다시 확인
• Testing 상태: Select Action → Suspend → 이후 Test Again
• Suspended 상태: Select Action → Test Again
→ 학생에게 다시 로그인하라고 안내한 후 Confirm 진행
❓ 잘못된 시험이 배정되었어요
아직 시험 시작 전 (To Be Confirmed): Select Action → Do Not Confirm → Assign Test로 재배정
이미 시험 중 (Testing): Select Action → Suspend (나중에 그 시험을 볼 수도 있으면) 또는 Terminate (그 시험을 완전히 취소) → Test Again → 올바른 시험 재배정
⚠️ Terminate는 모든 답안을 삭제하므로 신중하게 사용하세요.
❓ 문제 화면이 하얗게 나와요 (Blank Screen)
1. Proctor 콘솔에서 해당 학생 선택 → Select Action → Suspend
2. 학생 기기에서 테스트 브라우저 완전 종료
3. Proctor 콘솔에서 해당 학생 → Select Action → Test Again
4. 학생이 다시 로그인하면 중단된 곳부터 이어서 진행됩니다
❓ 시험이 너무 느려요
학생 기기에서 Reset 버튼을 클릭하거나, 키보드로 F5 (PC) 또는 Cmd+R (Mac)을 눌러 화면을 새로고침하세요.
❓ 결함이 있는 문제를 건너뛰고 싶어요
1. Proctor 콘솔에서 해당 학생 선택 → Select Action → Pause
2. 다시 학생 선택 → Select Action → Resume
3. 학생이 Resume을 누르면 새 문제가 표시됩니다

문제 자체를 신고하려면:
1. Proctor 콘솔에서 Proctor PIN 확인
2. 학생 기기에서 Ctrl+Shift+P 입력
3. PIN 입력 후 문제 상황 설명 → Resume Test
❓ "Students are Ineligible to Test" 메시지가 나와요
시험이 "학기당 1회"(Once Per Term)로 제한되어 있을 수 있습니다. District Assessment Coordinator에게 시험 제한 설정 확인을 요청하세요.
❓ 중단(Suspend)된 시험을 이어서 보려면?
같은 날, 같은 세션: 해당 학생 선택 → Select Action → Test Again

다른 날:
1. 저장된 세션을 다시 열거나 새 세션 생성
2. Find Students to Test → Test History Search 탭
3. 학교, 시험 등 필터 설정 → Search
4. 학생 추가 → 시험 배정 → Test Now
5. 확인(Confirm) 시 Resume Test 선택

⏰ 14일 이내에 재개하는 것을 권장합니다 (최대 28일 가능).

🌐 영상을 한국어 자막으로 보는 방법 (Chrome 브라우저)

NWEA 영상은 영어로 제공되지만, Chrome의 실시간 번역 자막 기능으로 한국어로 볼 수 있습니다:

  1. Chrome 브라우저에서 영상 링크 클릭
  2. 주소 표시줄 오른쪽의 ⋮ (더보기) → 설정 → 접근성
  3. "실시간 자막" 켜기 → "번역" 토글도 켜기
  4. 번역 언어를 한국어로 설정
  5. 영상을 재생하면 화면 하단에 한국어 자막이 실시간으로 표시됩니다

💡 한 번 설정하면 이후 모든 영상에 자동 적용됩니다. Chrome 103 이상 버전에서 지원됩니다.

🎬 Proctor 영상 시리즈 — 한글 요약 + 영상 링크
STEP 1 MAP Growth 로그인
MAP 관리 플랫폼(start.mapnwea.org)에 접속하여 로그인하는 방법을 안내합니다. 로그인 후 MAP Growth를 선택하고 Manage Test Sessions 메뉴로 이동하는 과정까지 보여줍니다.
▶ 영상 보기
STEP 2a Find Students to Test — 학생 검색
학교·학년·반 등 조건으로 학생을 검색하여 시험 세션에 추가하는 방법입니다. 학생 수는 100명 이하로 유지하고, 세션 이름은 공백 없이 짧게 만드는 팁도 포함됩니다.
▶ 영상 보기
STEP 2b Test My Class — 내 학급 시험
본인이 담당하는 학급을 바로 시험하는 가장 빠른 방법입니다. Test My Class 버튼 하나로 Proctor 콘솔이 바로 열리고, 학생들에게 시험을 배정할 수 있습니다.
▶ 영상 보기
STEP 3a 시험/편의사항 배정 (세션 편집 중)
시험 세션을 만들거나 편집하는 단계에서 학생에게 시험(예: Growth Math 6+)과 편의사항(Text-to-Speech 등)을 배정하는 방법을 보여줍니다.
▶ 영상 보기
STEP 3b 시험/편의사항 배정 (활성 세션)
이미 시험이 시작된 활성 세션에서 추가 학생에게 시험을 배정하거나, 편의사항을 변경하는 방법입니다. 시험 시작 전에만 변경 가능하다는 점에 유의하세요.
▶ 영상 보기
STEP 4 학생 Confirm (확인)
학생이 로그인하면 Proctor 콘솔에 "To Be Confirmed" 상태로 표시됩니다. Refresh Page → Confirm 버튼으로 학생을 확인하면 학생이 Start Test를 눌러 시험을 시작할 수 있습니다.
▶ 영상 보기
STEP 5 Pause & Resume (일시정지/재개)
시험 중 일시정지가 필요할 때 Pause를 사용합니다. 최대 25분간 유지되며, 같은 기기에서만 Resume 가능합니다. 25분 초과 시 자동으로 Suspend 상태로 전환됩니다.
▶ 영상 보기
STEP 6 Suspend & Test Again (중단/재시험)
장기 중단이 필요할 때 Suspend를 사용합니다. 최대 28일(14일 권장) 내에 Test Again으로 이어서 진행할 수 있습니다. 다른 기기, 다른 세션에서도 재개 가능합니다.
▶ 영상 보기
STEP 7 시험 완료 (Completed)
학생이 모든 문제에 답하면 "Completed" 상태가 됩니다. 완료된 학생에게 추가 시험을 배정하려면 Test Again → Assign Test로 새 시험을 지정합니다.
▶ 영상 보기
문제해결 Confirmed 상태 멈춤
학생이 Confirmed 상태에서 멈춰 시험이 시작되지 않는 경우의 해결 방법입니다. 네트워크 문제나 브라우저 이슈가 원인일 수 있으며, Do Not Confirm → 재로그인 → 재확인으로 해결합니다.
